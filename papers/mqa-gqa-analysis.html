MQA & GQA (Multi-Query / Grouped-Query Attention) 深度分析报告

MQA 论文：Fast Transformer Decoding: One Write-Head is All You Need — Noam Shazeer, Google, 2019

MQA 链接：https://arxiv.org/abs/1911.02150

GQA 论文：GQA: Training Generalized Multi-Query Transformer Models from Multi-Head Checkpoints — Joshua Ainslie, James Lee-Thorp, Michiel de Jong et al., Google Research, 2023

GQA 链接：https://arxiv.org/abs/2305.13245

一、问题背景：自回归推理的内存带宽瓶颈

Transformer 自回归解码（incremental inference）的核心瓶颈不在计算量，而在内存带宽。每生成一个 token，都需要：

  1. 从显存加载完整的 KV Cache（所有历史 token 的 key/value 向量）
  2. 计算当前 query 与所有 key 的注意力分数
  3. 用注意力权重加权 value
瓶颈所在：对于 MHA (Multi-Head Attention)，KV Cache 大小为 2 × n_layers × n_heads × seq_len × d_head。每个 token 生成需要将整个 KV Cache 从 HBM 加载一次。当序列长度和模型规模增大时，内存带宽成为解码速度的决定因素。

二、MQA：极致压缩——所有 head 共享一份 K/V

2.1 核心思想

标准 MHA 中，每个 attention head 有独立的 Q/K/V 投影。MQA 的改变：保持 Q 的 H 个独立头，但所有头共享同一个 K 和同一个 V。

Multi-Head Attention (MHA): Q: [batch, n_heads, seq, d_head] ← H 个独立的 query 头 K: [batch, n_heads, seq, d_head] ← H 个独立的 key 头 V: [batch, n_heads, seq, d_head] ← H 个独立的 value 头 Multi-Query Attention (MQA): Q: [batch, n_heads, seq, d_head] ← H 个独立的 query 头（不变） K: [batch, 1, seq, d_head] ← 所有头共享 1 个 key V: [batch, 1, seq, d_head] ← 所有头共享 1 个 value KV Cache 大小压缩比：H 倍（H 通常为 32-128）

2.2 性能分析

维度MHAMQA
KV Cache 大小2 × H × seq × d_head2 × 1 × seq × d_head
推理带宽需求高（每步加载全部 KV）低（仅加载 1/H 的 KV）
模型表达能力最强（每头独立 K/V 表示）受限（所有 query 看到相同 K/V）
训练时间基线基本相同（计算量差异小）
推理速度基线显著加速（memory-bound 场景）

2.3 MQA 的理论分析

MQA 论文给出了关键的 roofline 分析：

批量推理（parallel）： 算术运算量：Θ(b·n·d²) 内存访问量：O(b·n·d + b·H·n² + d²) 比率（算术强度）：高 → compute-bound → MQA 无显著帮助 增量解码（incremental, n=1）： 算术运算量：Θ(b·d²) 内存访问量：O(b·n·d·H + d²) ← KV Cache 主导! 比率（算术强度）：极低 → memory-bound → MQA 带来 H 倍带宽减少 结论：MQA 的加速仅在增量解码（自回归生成）时显著。

三、GQA：平衡的中间方案

3.1 MQA 的问题

  • 质量下降：所有 query head 共享同一份 K/V，表达能力受限
  • 训练不稳定：某些场景下 MQA 导致训练更不稳定
  • 已有模型不兼容：许多已训练的 MHA 模型（如 LLaMA-1、T5）无法直接转为 MQA

3.2 GQA 设计

Grouped-Query Attention (GQA-G)： 将 H 个 query head 分为 G 组 每组共享 1 个 key head 和 1 个 value head 特殊情况： GQA-1 = MQA（1 组，即所有头共享） GQA-H = MHA（H 组，即每头独立） GQA-G（1 < G < H）= 折中方案 例如 LLaMA-2 70B： H = 64 query heads, G = 8 key-value groups 每 8 个 query head 共享 1 个 KV head KV Cache 压缩比 = 64/8 = 8x

3.3 Uptraining：从 MHA 模型转换

GQA 论文的另一关键贡献是无需从头训练的转换方法：

  1. 检查点转换：将同一组内的多个 K/V 投影矩阵取平均（mean pooling），得到该组的共享 K/V 投影
  2. 少量继续预训练：在原始预训练数据上继续训练 α 比例的步数（α=5% 即足够）

这使得已有的 MHA 大模型可以低成本转换为 GQA 模型获得推理加速。

五、关键问答

Q1：为什么推理时 KV Cache 的带宽消耗比模型权重更关键？

A：模型权重在所有 token 之间共享，可以驻留在 GPU 显存中复用。但 KV Cache 随序列长度线性增长——对于 70B 模型、8K 序列、FP16，KV Cache 可达 20+ GB。自回归解码每步仅做一个 token 的计算（算术强度极低），但需要加载整个 KV Cache 做注意力。这使得运算几乎完全受带宽限制。减少 KV Cache 大小（通过共享 K/V head）直接转化为解码速度提升。

Q2：MQA 的质量下降来自哪里？为什么 GQA 能缓解？

A：MHA 中每个 head 学习关注输入的不同方面（不同的 K/V 子空间）。MQA 强制所有 head 共享 K/V，意味着所有 head 必须在同一个 key-value 空间中找到自己需要的信息。这限制了模型的多样性——不同 head 无法"看到"输入的不同侧面。GQA 的分组方案保留了部分多样性：G 组意味着 G 个不同的 K/V 子空间，在完全共享（MQA）和完全独立（MHA）之间找到了甜点。

Q3：为什么用 mean pooling 而非选择单个 head 或随机初始化？

A：实验比较了三种 checkpoint 转换策略：(1) 随机选择组内的一个 head；(2) 随机初始化新的共享 head；(3) 对组内所有 head 做 mean pooling。Mean pooling 效果最好，原因是它保留了组内所有 head 已学习信息的"平均共识"——相当于用 ensemble 的思想初始化。选择单个 head 会丢失其他 head 的知识；随机初始化则完全抛弃已学习信息。

Q4：GQA 的 G 值如何选择？

A：G 的选择需要考虑：(1) 模型规模——更大的模型（更多 head）可以承受更激进的压缩；(2) 目标硬件的并行度——G 应整除 tensor parallel 的分片数，避免通信不均；(3) 经验值——LLaMA-2 70B 使用 G=8（H=64），Mistral 使用 G=8（H=32）。通常 G=4~8 是常见选择，在质量损失 <1% 的前提下获得 4-8x KV Cache 压缩。

Q5：MQA/GQA 与 FlashAttention 如何配合？

A：二者优化不同维度的瓶颈。FlashAttention 优化训练和 prefill 阶段的 IO（减少 N×N 注意力矩阵的 HBM 读写）；MQA/GQA 优化自回归解码阶段的 KV Cache 带宽。二者完全正交且互补——现代 LLM 同时使用两者：用 FlashAttention 加速训练和长上下文 prefill，用 GQA 加速 token-by-token 生成。

六、总结与后续影响

MQA 贡献

首次明确识别自回归推理的内存带宽瓶颈，提出共享 K/V 的极简方案。PaLM、StarCoder 等模型直接采用。

GQA 贡献

提供了 MHA 到 MQA 的连续插值方案；提出低成本 uptraining 方法使已有模型可转换。被 LLaMA-2/3、Mistral 等采用。

设计启示

推理效率优化的关键是减少 memory-bound 阶段的数据移动量。K/V 的冗余度远高于 Q——共享 K/V 的质量损失远小于共享 Q。

后续发展

启发了 MLA（DeepSeek-V2 的低秩 KV 压缩）、KV Cache 量化、PagedAttention（vLLM）等进一步的推理优化方向。
