二、摘要与核心贡献剖析

本文提出 DeepSeekMath 7B 模型，在竞赛级 MATH 基准上达到 51.7% 正确率（接近 GPT-4 水平），其核心技术贡献包括：(1) 从 Common Crawl 中筛选 120B 高质量数学 token 的数据管线；(2) GRPO——一种高效的 PPO 变体，去掉 value model 节省训练资源。本报告聚焦 GRPO 算法。

GRPO 的核心设计思想：
  • PPO 需要训练一个与 policy 同等规模的 value model 来估计 advantage——在 LLM 场景中开销巨大
  • GRPO 的替代方案：对同一 prompt 采样 G 个输出，用组内奖励的均值和标准差来归一化，归一化后的奖励直接作为 advantage
  • 这种"组内相对"的设计与 reward model 的训练方式（同一问题多个输出的相对比较）天然一致

三、背景知识补充

3.1 PPO 在 LLM 微调中的角色

PPO (Proximal Policy Optimization) 是 InstructGPT/ChatGPT 采用的 RL 算法。在 LLM 场景中，PPO 需要同时维护四个模型：

模型作用参数量
Policy Model π_θ当前策略（被优化的模型）与目标 LLM 相同
Reference Model π_refKL 约束的锚点（冻结）与目标 LLM 相同
Reward Model r_φ评估输出质量（冻结）通常为 LLM 规模
Value Model V_ψ估计每个 token 的期望累计奖励（需训练）与目标 LLM 相同

Value Model 的问题：(1) 显存翻倍——多了一个与 policy 同等规模的可训练模型；(2) 训练困难——数学场景中只有最后一个 token 获得 reward，中间 token 的 value 估计不稳定。

3.2 Advantage 的含义

Advantage A_t = Q(s_t, a_t) - V(s_t)，即在状态 s_t 下采取动作 a_t 相比"平均水平"好多少。在 PPO 中用 GAE (Generalized Advantage Estimation) 计算，需要 value model 提供 V 估计。GRPO 的创新在于不用 value model，而是直接从同一 prompt 的多个采样中估计"平均水平"。

4.3 Advantage 估计：Outcome Supervision

对每个问题 q，采样 G 个输出 {o_1, ..., o_G} Reward model 评分得到 {r_1, ..., r_G} 归一化：r_tilde_i = (r_i - mean(r)) / std(r) Advantage：A_hat_{i,t} = r_tilde_i（对 o_i 中所有 token 相同） 含义： - 如果 o_i 的奖励高于组内平均 → advantage > 0 → 增大 o_i 的概率 - 如果 o_i 的奖励低于组内平均 → advantage < 0 → 减小 o_i 的概率 - 标准差归一化确保梯度幅度不受奖励绝对值影响

4.4 Advantage 估计：Process Supervision

对每个输出 o_i 的每个推理步骤 j，process reward model 给出 r^{index(j)}_i 归一化：跨所有 G 个输出的所有步骤奖励一起计算 mean 和 std Advantage：A_hat_{i,t} = Σ_{index(j) >= t} r_tilde^{index(j)}_i （从当前 token 之后所有步骤的归一化奖励之和） 相比 Outcome Supervision：提供更细粒度的信号，但需要训练 process reward model

4.6 迭代 GRPO (Iterative GRPO)

为应对 reward model 在策略进步后可能过时的问题，论文提出迭代训练：

  1. 用当前策略采样新输出，构建新的 reward model 训练集
  2. 持续训练 reward model（加入 10% 历史数据防止遗忘）
  3. 将当前策略设为新的 reference model
  4. 用新 reward model 继续训练策略

七、关键问答

Q1：为什么用组内均值替代 value model 是合理的？

A：Value model V(s_t) 的本质是"在当前状态下，按照当前策略继续下去，期望能获得的总奖励"。对于整个输出级别的 outcome supervision，这等价于"对同一 prompt 的平均奖励"。GRPO 通过对同一 prompt 采样 G 个输出并取均值，直接用蒙特卡洛方法估计了这个期望。当 G 足够大（论文中 G=64），估计精度足够替代训练一个参数化的 value model。这是用计算换内存的权衡——多采样 64 次来避免训练一个 7B 参数的 value model。

Q2：GRPO 的 KL 惩罚为什么要从 reward 中分离出来？

A：PPO 中 KL 惩罚嵌入 reward：r_t = r_φ - β·log(π/π_ref)，这导致 advantage 计算时需要 value model 同时学习"真实奖励"和"KL 惩罚"的混合信号，增加训练难度。GRPO 将 KL 作为独立正则项加入损失函数，优势有二：(1) advantage A_hat 仅反映输出质量的相对好坏，信号更纯净；(2) KL 正则项可以独立控制强度，不干扰组内奖励的归一化。

Q3：G=64 的组大小是否太大？对计算开销有何影响？

A：G=64 意味着每个训练 step 需要对每个 prompt 生成 64 个完整输出。这确实带来大量推理开销，但与 PPO 的 4 模型方案相比仍然更高效——节省了一整个 7B value model 的显存和训练成本。实际工程中，生成阶段可以高度并行（大 batch 推理），而且每次 exploration 后策略只更新一次（low update-to-data ratio），进一步减少训练步数。

Q4：GRPO 与 DPO 相比，本质区别是什么？

A：在论文提出的统一框架下，二者都是偏好优化方法，但有关键差异：
(1) 在线 vs 离线：GRPO 是在线方法（每步从当前策略采样），DPO 是离线方法（使用固定的偏好数据集）。在线方法能持续获得当前策略的分布信息，避免分布偏移。
(2) 奖励信号：GRPO 使用显式 reward model 评分，DPO 使用隐式奖励（策略本身定义奖励）。
(3) 优化粒度：GRPO 可以做 token-level 优化（process supervision），DPO 是 sequence-level。
论文实验表明在数学推理任务中，在线 RL（GRPO）优于离线优化（DPO）。

Q5：为什么 GRPO 的 RL 训练能在域外任务上也获得提升？

A：论文推测 RL 训练不仅教会模型"如何回答数学题"，更强化了底层的推理模式——如逻辑链路的严谨性、步骤之间的因果连贯性。这些元能力可以迁移到其他需要结构化推理的任务（如中文数学、工具调用）。这与 InstructGPT 中"跟随指令"能力泛化到非英语任务的现象类似——RL 对齐的是一种通用的认知模式，而非特定任务的表面形式。

八、总结与后续影响

算法创新

用组内相对奖励替代 value model 进行 advantage 估计，将 PPO 的 4 模型方案简化为 3 模型（policy + reference + reward），显存节省约 25-40%。

理论统一

提出了统一框架理解 RFT、DPO、PPO、GRPO——它们都是直接或简化的 RL 技术，差异在于 on/off-policy、advantage 估计方式和更新频率。

工程实践

G=64 采样 + 单步更新 + β=0.04 KL 约束的配置被后续工作广泛复用。证明了即使在较小模型 (7B) 上，RL 也能带来显著推理提升。

后续影响

GRPO 被 DeepSeek-R1、DeepSeek-V2.5 等直接采用；启发了其他无 value model 的 RL 算法；证明了数学推理是 RL for LLM 的理想测试场景。
