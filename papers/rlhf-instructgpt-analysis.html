RLHF / InstructGPT 深度分析报告

论文标题：Training language models to follow instructions with human feedback

论文链接：https://arxiv.org/abs/2203.02155

作者团队：Long Ouyang, Jeff Wu, Xu Jiang, Diogo Almeida, Carroll L. Wainwright, Pamela Mishkin, Chong Zhang, Sandhini Agarwal, Katarina Slama, Alex Ray, John Schulman, Jacob Hilton, Fraser Kelton, Luke Miller, Maddie Simens, Amanda Askell, Peter Welinder, Paul Christiano, Jan Leike, Ryan Lowe

关联机构：OpenAI

发表时间：2022年3月

一、论文基本信息

项目内容
研究问题如何让语言模型的行为与人类意图对齐（aligned），使其输出更有帮助、更真实、更少有害？
核心方法三阶段 RLHF 流程：SFT（监督微调）→ RM（奖励模型训练）→ PPO（策略优化）
基线模型GPT-3（1.3B / 6B / 175B 参数量），架构不变
关键结论1.3B InstructGPT 在人类评估中优于 175B GPT-3，参数量少 100 倍；RLHF 的对齐成本远低于预训练成本
历史地位首次在生产级语言模型上系统验证 RLHF 对齐方法，成为 ChatGPT 等产品的技术基础

二、摘要与核心贡献剖析

本文的核心洞察是：语言建模目标（预测下一个 token）与"遵循用户指令"这一目标之间存在本质错位。仅仅扩大模型规模并不能修复这种错位。论文提出了一种基于人类反馈的强化学习（RLHF）方法来弥合这一鸿沟。

核心贡献：
  • 建立了完整的 RLHF 三阶段对齐流程（SFT → RM → PPO），并在 GPT-3 上验证有效性
  • 证明了对齐微调的性价比极高：1.3B 对齐模型 > 175B 未对齐模型
  • 提出 PPO-ptx 变体解决"对齐税"问题（alignment tax）——在公共 NLP 基准上的性能退化
  • 发现对齐能力可泛化到训练分布之外（非英语任务、代码任务）

三、背景知识补充

3.1 预训练语言模型的对齐问题

预训练语言模型（如 GPT-3）的训练目标是最大化 Internet 文本上的下一个 token 概率。这个目标本质上是"模仿网页文本的统计分布"，而非"服务于用户意图"。二者的错位表现为：

  • 有害输出：训练数据包含有毒内容，模型会复现
  • 幻觉（Hallucination）：模型会自信地编造事实
  • 不遵循指令：模型倾向于"续写"而非"执行指令"

3.2 强化学习基础：PPO 算法

PPO（Proximal Policy Optimization）是 OpenAI 提出的策略梯度算法，其核心思想是限制每次策略更新的幅度，通过 clip 操作保证训练稳定性。在 RLHF 场景中，PPO 被用于将语言模型（策略 π）的生成行为调整为最大化奖励模型给出的分数。

3.3 从 RLHF for Summarization 到 InstructGPT

RLHF 的方法论继承自 Stiennon et al. (2020) 在文本摘要任务上的工作。InstructGPT 的关键扩展是将该方法从单一任务推广到广泛的指令跟随任务分布，包括生成、问答、对话、摘要、改写等多种场景。

四、方法与算法原理

4.1 总体流程：三阶段 RLHF

阶段输入输出方法
Step 1: SFT标注员撰写的高质量回复示范监督微调模型 πSFT标准监督学习，16 epochs，cosine LR decay，dropout 0.2
Step 2: RM标注员对模型输出的排序偏好奖励模型 rθBradley-Terry 模型 + 批次内所有 C(K,2) 对比较
Step 3: PPOAPI 提示词，奖励模型信号对齐策略 πRLPPO + KL 惩罚 + 可选预训练梯度混合

4.2 奖励模型（Reward Model）设计

奖励模型的核心设计决策：

  • 架构：基于 SFT 模型，移除最后的 unembedding layer，替换为标量输出头
  • 模型大小：仅使用 6B 参数（而非 175B），因为 175B RM 训练不稳定，且 6B 足够节省计算
  • 标注效率优化：每次展示 K=4~9 个回复让标注员排序，产生 C(K,2) 对比较

RM 损失函数

loss(θ) = -1/C(K,2) * E_{(x,y_w,y_l)~D} [log(σ(r_θ(x, y_w) - r_θ(x, y_l)))] 其中： r_θ(x, y) = 奖励模型对 prompt x 和 completion y 的标量输出 y_w = 标注员偏好的（winning）回复 y_l = 不被偏好的（losing）回复 σ = sigmoid 函数
关键设计：将每个 prompt 的所有 C(K,2) 对比较作为单个 batch element 训练。这样每个 completion 只需一次前向传播，既高效又避免过拟合（因为 comparisons 在同一 labeling task 内高度相关）。

4.3 PPO 强化学习阶段

RL 阶段的环境是一个 bandit environment：随机给出 prompt，模型生成 response，奖励模型给出分数，episode 结束。

PPO-ptx 目标函数（完整版）

objective(φ) = E_{(x,y)~D_π^RL} [r_θ(x,y) - β·log(π^RL_φ(y|x) / π^SFT(y|x))] + γ·E_{x~D_pretrain} [log(π^RL_φ(x))] 组成部分： 第一项：奖励模型分数（越高越好） 第二项（负号）：KL 散度惩罚，防止策略偏离 SFT 模型过远 第三项：预训练分布上的 log-likelihood（缓解对齐税） 超参数： β = KL 惩罚系数 γ = 预训练混合系数（PPO 模型中 γ=0，PPO-ptx 模型中 γ>0）

4.4 KL 散度惩罚的作用

per-token KL 惩罚 β·log(π^RL/π^SFT) 有两个关键作用：

  1. 防止奖励模型过度优化（reward hacking）：没有 KL 约束，策略会学习"钻空子"——产生奖励模型给高分但人类不喜欢的退化输出
  2. 保持语言能力：确保模型不会为了追求高奖励而丧失基本的语言生成质量

4.5 对齐税与 PPO-ptx 解决方案

纯 PPO 微调导致在 SQuAD、DROP、HellaSwag 等公共基准上出现性能退化。这被称为对齐税（alignment tax）——对齐模型需要付出的能力代价。

PPO-ptx 的解决方案：在 PPO 梯度更新中混入预训练分布上的梯度（目标函数中的 γ 项）。实验证明这比简单增大 KL 系数更有效——后者在恢复基准性能的同时会显著降低奖励得分。

六、实验结果

6.1 核心对比结果

模型vs. 175B SFT 胜率关键特征
GPT-3 175B~22%基线，不遵循指令
GPT-3 175B (prompted)~29%精心设计的 few-shot prefix 改善指令跟随
SFT 175B50%（基线）监督微调基线

6.2 具体改善维度

  • 真实性：TruthfulQA 上真实且有信息量的回复比 GPT-3 多约一倍
  • 幻觉：封闭域任务幻觉率从 41%（GPT-3）降至 21%（InstructGPT）
  • 毒性：使用"respectful"指令时毒性输出减少约 25%
  • 指令遵循：更可靠地遵守显式约束（如字数限制、格式要求）
  • 偏见：Winogender/CrowS-Pairs 上未见明显改善

6.3 泛化能力

InstructGPT 展现了超越训练分布的泛化：能够跟随非英语指令（训练数据 96%+ 为英语）、回答代码相关问题。这表明 RLHF 不仅教会模型特定任务的回答方式，还教会了"跟随指令"这一元能力。

七、关键问答

Q1：为什么 1.3B InstructGPT 能胜过 175B GPT-3？这是否说明对齐比规模更重要？

A：这说明在特定任务维度（指令遵循、有用性）上，目标对齐的价值远超纯粹的规模增长。GPT-3 175B 拥有强大的世界知识和语言能力，但其优化目标（next-token prediction）不直接服务于用户指令。RLHF 将模型的行为与用户意图对齐，等效于在已有能力基础上安装了一个"意图导航系统"。但需注意，这并非说对齐可以完全替代规模——在知识密集型任务上，大模型的优势仍然存在。

Q2：为什么奖励模型只用 6B 而不用 175B？

A：两个原因：(1) 计算效率——RM 在 PPO 训练过程中每个 token 都需要前向传播，小模型显著降低开销；(2) 训练稳定性——175B RM 训练不稳定（unstable），不适合作为 RL 中的 value function。这暗示了一个重要设计原则：RM 不需要与策略模型同等规模，因为"判断输出好坏"比"生成高质量输出"更容易。

Q3：为什么用批次内 C(K,2) 对比较而非独立 pair？

A：同一 prompt 的 K 个回复之间的 pairwise comparison 高度相关（共享相同 prompt 和标注员偏好倾向）。如果将这些 pair 打散成独立数据点，每个 completion 会在一个 epoch 内被用于 K-1 次梯度更新，导致严重过拟合。将所有 C(K,2) 对作为一个 batch element 训练：(1) 每个 completion 只需一次前向传播——计算效率提升 K/2 倍；(2) 在同一 batch 内看到所有相关比较，梯度信号更一致，避免过拟合。

Q4：KL 惩罚如何防止 reward hacking？其 β 值如何选择？

A：奖励模型是对人类偏好的不完美近似。如果不约束策略，它会找到 RM 的"盲区"——产生获得高分但实际低质量的输出（类似 adversarial example）。KL 惩罚限制策略不能偏离 SFT 模型太远，确保输出仍保持语言质量。β 的选择是 reward score 和输出质量之间的权衡：β 太小导致 reward hacking，β 太大导致策略无法学习新行为。论文未公开 β 具体值，但从实验结果看选择了使验证奖励与输出质量同时最优的平衡点。

Q5：PPO-ptx 为什么比增大 KL 系数更有效地解决对齐税？

A：增大 KL 系数是"间接"保持通用能力——它限制策略偏离的幅度，但不能保证保留哪些能力。PPO-ptx 是"直接"在预训练目标上提供监督信号，明确告诉模型"保持在通用文本上的建模能力"。实验表明增大 KL 系数导致 validation reward 显著下降且无法完全恢复 DROP/SQuAD 性能，而 PPO-ptx 可以找到一个混合系数 γ 同时恢复基准性能并保持高验证奖励。

Q6：RLHF 的对齐成本（60 PF-days）相比预训练（3640 PF-days）意味着什么？

A：这是论文最重要的实践结论之一：对齐的计算成本不到预训练的 2%，但带来的用户偏好提升超过 100 倍规模增长。这为产业界提供了强力论据——与其投入更多算力训练更大的模型，不如在现有模型上投入少量资源进行对齐。这一结论直接影响了后续 ChatGPT、Claude 等产品的技术路线选择。

八、总结与后续影响

方法论贡献

建立了 SFT → RM → PPO 三阶段 RLHF 标准流程，成为后续所有对齐工作的基础范式。证明该方法可从摘要等单任务扩展到通用指令跟随。

工程实践

揭示了关键工程选择：6B RM 足够、批次内比较训练、PPO-ptx 缓解对齐税、per-token KL 惩罚。这些设计被后续工作广泛采用。

局限性

对齐目标是特定标注团队的偏好，非"通用人类价值"；模型仍然会幻觉、被假前提误导、过度 hedge；偏见问题未显著改善。

后续影响

直接催生 ChatGPT；激发 DPO 等无需 RL 的对齐方法；RLHF 成为 LLM 产品化的标准步骤；推动了 Constitutional AI、RLAIF 等改进方向。
